1. **Добавлено поле `shortName` в `CompetitionEntity`**
   - **Дата:** 24.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** Полное название «Бяковские каменоломни 2026» слишком длинное для статус-бара (48dp высоты). Экономия места на экране. В протоколах используется полное `name`.
   - **Файлы:** `CompetitionEntity.kt`, `CompetitionNewScreen.kt`, `CompetitionNewVM.kt`

2. **Добавлено поле `nickname` (позывной) в `PersonEntity`**
   - **Дата:** 24.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** В спелеосообществе принято использовать позывные. Полезно для быстрой идентификации.
   - **Файлы:** `PersonEntity.kt`, `PersonNewScreen.kt`

3. **Цветовые метки команд заменены с эмодзи 🟢🟠🔴 на текст**
   - **Дата:** 24.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** Эмодзи по-разному отображаются на разных устройствах. Текст читается однозначно.
   - **Было:** `🟢 Зелёная метка`
   - **Стало:** `18+`, `14+`, `<14ок`, `<17 !!!`, `<14 !!!`, `СНЯТЫ`

4. **Метка «ДИСКВ» заменена на «СНЯТЫ»**
   - **Дата:** 24.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** «СНЯТЫ» — более понятный русский термин, чем сокращение «ДИСКВ».

5. **«⚠️ Разрешено судьёй» — цвет изменён с красного на тёмно-зелёный**
   - **Дата:** 24.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** Красный цвет ассоциируется с ошибкой. Разрешение — это наоборот «всё в порядке». Тёмно-зелёный `#2E7D32`.

6. **Аватары: свои SVG вместо эмодзи 👤👦👧**
   - **Дата:** 24.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** Эмодзи выглядят по-разному на Android и iOS. SVG гарантирует единый стиль.
   - **Файлы:** `ic_avatar_male.svg` (голубой), `ic_avatar_female.svg` (розовый), `ic_avatar_default.svg` (серый)

7. **Таймер: формат `ММ:СС.сс`, часы мельче если `00`**
   - **Дата:** 25.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** Экономия места, пока соревнование не перевалило за час. Сотые важны для финиша.

8. **Время финиша фиксируется при открытии диалога, а не при подтверждении**
   - **Дата:** 23.04.2026
   - **Инициатор:** Разработчик (Fakel) + обсуждение
   - **Причина:** Пока судья выбирает команды в списке, время не должно «тикать». Это справедливо для группового финиша.

9. **Отсечка вычитается из времени, даже если КП «взят с ошибкой»**
   - **Дата:** 22.04.2026
   - **Инициатор:** Разработчик (Fakel)
   - **Причина:** Команда всё равно ждала в очереди и потратила время. Справедливо.

10. **Смена класса разрешена первые 5 минут после старта со штрафом**
    - **Дата:** 22.04.2026
    - **Инициатор:** Разработчик (Fakel)
    - **Причина:** Реальный кейс: команда поняла, что ошиблись классом, но ещё не ушла далеко. Штраф настраивается в настройках.

11. **Кнопка «Экспорт сейчас» на главном экране**
    - **Дата:** 24.04.2026
    - **Инициатор:** Разработчик (Fakel)
    - **Причина:** Страх судей потерять данные. Быстрый доступ без перехода в настройки.

12. **Автоэкспорт в фоне при каждом изменении**
    - **Дата:** 24.04.2026
    - **Инициатор:** Обсуждение
    - **Причина:** Если приложение крашнется — данные не потеряются. Файл `autobackup.json` ~300 КБ.

13. **Импорт в «голое» приложение**
    - **Дата:** 24.04.2026
    - **Инициатор:** Разработчик (Fakel)
    - **Причина:** Сценарий: телефон сломался, флешка → новый телефон → восстановление.

14. **Таймер восстанавливается через БД, а не SharedPreferences**
    - **Дата:** 25.04.2026
    - **Инициатор:** Обсуждение (вариант Б)
    - **Причина:** SharedPreferences может сброситься при очистке кэша. БД надёжнее.

15. **Порядок очистки БД: дети → родители**
    - **Дата:** 25.04.2026
    - **Инициатор:** Сторонний наблюдатель
    - **Причина:** `FOREIGN KEY constraint` при удалении родительской таблицы раньше дочерней.

16. **Генератор тестовых данных обёрнут в `db.withTransaction`**
    - **Дата:** 25.04.2026
    - **Инициатор:** Сторонний наблюдатель
    - **Причина:** Атомарность: если генерация упадёт на середине — не останется «мусора» в БД.

17. **Версия БД повышается при ЛЮБОМ изменении Entity**
    - **Дата:** 25.04.2026 (выстрадано)
    - **Инициатор:** 4-часовой баг `FOREIGN KEY`
    - **Причина:** Старые данные + новая схема = крах. Всегда `version++`.

18. **Кнопки "+" и "✖" внутри поля поиска**
    - **Дата:** 03.05.2026
    - **Инициатор:** Разработчик (Fakel)
    - **Причина:** При поиске персоны нельзя было создать новую, если в выдаче были однофамильцы. Кнопка создания была скрыта за результатами.
