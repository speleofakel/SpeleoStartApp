# СпелеоСтарт (SpeleoStart)

Приложение для судейства соревнований по подземному ориентированию.
Работает полностью офлайн в пещерах и каменоломнях.

## Быстрый старт

1. Установите APK на планшет/телефон (Android 7.0+)
2. Запустите приложение
3. Создайте соревнование (**Соревнования** → **➕ Новое**)
4. Добавьте персон (**Персоны** → **➕ Новая**)
5. Зарегистрируйте команды (**Регистрация**)
6. Начинайте старты (**Старт** → **НАЧАТЬ СОРЕВНОВАНИЯ**)

## Основные функции

- Регистрация команд и участников (личный/командный зачёт)
- Автоматическое чередование классов на старте (`2→3→2→3`)
- Таймер с восстановлением после краша/перезагрузки
- Фиксация финиша с ручной корректировкой времени
- Электронные путевые листы (ПЛ) с двойным подтверждением
- Автоматический подсчёт баллов и итогов
- Экспорт протоколов (JSON/CSV) и генерация грамот (PDF)
- Поддержка произвольного состава команды (ОБГ)

## Технический стек

- **Kotlin** 1.9.22 + **Jetpack Compose** (BOM 2024.02.00)
- **Room** 2.6.1 (SQLite)
- **Hilt** 2.48 (DI)
- **Navigation Compose** 2.7.7
- **Coroutines + Flow** (асинхронность)
- **Timber** 5.0.1, **Security Crypto** 1.1.0

## Разработка

### Отладка (Debug-режим)
- Долгий тап по плитке **«Настройки»** → меню разработчика
- **«Заполнить тестовыми»** — создаёт соревнование, 20 персон, 10 команд
- **«Очистить тестовые»** — удаляет все данные (соблюдая порядок `дети → родители`)
- Таймер в debug-режиме работает ускоренно (`2 сек = 1 сек реального времени`)

## Важные правила разработки

1. При изменении `Entity` **ВСЕГДА** повышайте `version` в `AppDatabase`.
2. Порядок очистки БД: дочерние таблицы → родительские (`participants` перед `teams`).
3. Не хардкодить `competitionId`/`teamId` — используйте `SharedState`/ViewModel.
4. Всегда проверяйте результат `insert()` на `-1L`.
