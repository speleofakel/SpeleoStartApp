# ЛОГИКА ПРОЦЕССОВ · СпелеоСтарт

---

## 1. РЕГИСТРАЦИЯ КОМАНДЫ

```text
НАЧАЛО
│
▼
Ввод фамилии (3+ символов)
│
├──[источник "База персон"]──▶ Поиск в persons
│   ├──[найдено]──▶ Выпадающий список → выбор → автозаполнение
│   └──[не найдено]──▶ Ручной ввод
│
└──[источник "Словарь"]──▶ Поиск в names_dictionary.json
    ├──[найдено]──▶ Список фамилий → выбор → автоподстановка имени
    └──[не найдено]──▶ Ручной ввод
```

### Проверка возраста и ментора
```text
Введена дата рождения
│
▼
Расчёт возраста
│
├──[≥18]──▶ ✅ Ментор не требуется
├──[14-17]──▶ ⚠️ Требуется ментор (подтверждение)
├──[<14]──▶ ⚠️ Ментор + разрешение судьи
└──[некорректна]──▶ ❌ Ошибка
```

### Сохранение
```text
Нажата "СОХРАНИТЬ"
│
▼
Проверки:
├──[командный]──▶ мин. 2 участника
├──[личный]─────▶ мин. 1 участник
├──[ФИО заполнены?]
└──[обязательные поля?]
│
▼
✅ Создание: номер (сквозной в классе), статус registered, цветовая метка
```

## 2. СТАРТ

### Формирование очереди
```text
Получить все registered команды
│
▼
Разделить по классам: class2_list, class3_list
│
▼
Сортировать по team_number
│
▼
Чередование: 2 → 3 → 2 → 3 ...
```

### Старт команды
```text
Нажата "СТАРТ" или обратный отсчёт = 0
│
▼
Проверки:
├──[команда существует?]
├──[статус = registered?]
├──[мин. участников?]
└──[менторы для <18?]
│
▼
✅ Статус → started
start_time = System.currentTimeMillis()
Удалить из очереди
countdown = start_interval
```

### Пропуск
```text
Нажата "ПРОПУСК"
│
├──[первая команда?]──▶ ❌ Нельзя
└──[команда существует?]
│
▼
✅ skip_count++
Перенести в конец очереди своего класса
countdown = start_interval
```

### Откат старта
```textДолгий тап → "Откат старта"
│
├──[единственная started?]──▶ Сброс таймера, статус → registered, кнопка "НАЧАТЬ"
└──[есть другие started?]──▶ Статус → registered, возврат в конец очереди
```

## 3. ФИНИШ
```text
Открыт диалог финиша (FAB 🏁)
│
▼
finishTimestamp = System.currentTimeMillis()  ← ФИКСАЦИЯ СРАЗУ
│
▼
Показать все started команды (2 колонки)
│
▼
Выбор команд → "ПОДТВЕРДИТЬ"
│
▼
Для каждой выбранной:
finishTimestamp = зафиксированное время
Статус → finished
```

### Ручная корректировка
```text
Долгий тап по времени
│
▼
Диалог: часы, минуты, секунды
│
▼
Лог: "Время скорректировано. Судья: [ФИО]. Исходное: X, Новое: Y"
```

## 4. ПУТЕВОЙ ЛИСТ

### Заполнение КП
```text
Для каждого КП:
│
├──[обычный]──▶ Чекбоксы "Взят" / "Ошибка"
│   ├──[☑️ Взят, ☐ Ошибка]──▶ Вес засчитывается
│   ├──[☑️ Взят, ☑️ Ошибка]──▶ Вес НЕ засчитывается
│   └──[☐ Взят]─────────────▶ Вес НЕ засчитывается│
└──[технический]──▶ Дополнительно: Отсечка (ММ:СС), Штраф (баллы)
```

### Подтверждение
```text
Секретарь заполнил → пароль → "☑️ Карту заполнил"
Судья проверил → пароль → "☑️ Карту проверил"
│
▼
Оба отмечены → checkpointsEntered = true → read-only
```

### Пересчёт lost → finished
```text
После сохранения ПЛ:
│
▼
чистое_время = (финиш - старт) - Σ(отсечка)
│
▼
Если статус = lost И чистое_время ≤ КВ:
│
▼
✅ Статус → finished
Лог: "Статус изменён с lost на finished после учёта ожидания"
```

## 5. ИТОГИ

### Расчёт баллов
`Σ(баллы) = Σ(вес_КП, где taken=1 И error=0) - Σ(штрафы)`

### Ранжирование
```text
Сортировка:
Статус: finished > lost > disqualified
Баллы: больше → выше
Чистое время: меньше → выше
```

### Деление мест
```text
Если баллы равны И разница времени ≤ допуск:
│
▼
Предложить судье объединить
│▼
final_place = одинаковый
place_merged = true
```

## 6. РАСФОРМИРОВАНИЕ
```text
Долгий тап → "Расформировать"
│
├──[registered]──▶ Без пароля → участники → free_agent
├──[started]─────▶ Пароль + причина → возврат в registered → free_agent
└──[finished/lost/disqualified]──▶ ❌ Запрещено
```

---

## 7. СМЕНА КЛАССА
```text
│
├──[registered]──▶ Можно. Если в очереди → выбор: оставить/в конец/удалить
├──[started ≤5 мин]──▶ Пароль + штраф (настраивается)
├──[started >5 мин]──▶ ❌ Только откат или дисквалификация
└──[finished]────────▶ ❌ Только дисквалификация
```

## 9. ПРОВЕРКИ И ОГРАНИЧЕНИЯ (СВОДКА)

| Действие | Условие | Результат |
|----------|---------|-----------|
| Старт | ≥ мин. участников | ❌ «Недостаточно участников» |
| Старт | Все участники active | ❌ «Есть неактивные» |
| Пропуск | Не первая команда | ❌ «Нельзя пропустить первую» |
| Расформирование | registered или started | ❌ «Нельзя расформировать» |
| Финиш | Статус started | ❌ «Не на дистанции» |
| Подтверждение ПЛ | Не взаимоисключающие | ❌ «Нельзя отметить оба» |
| Финализация | Все finished имеют ПЛ | ❌ «Есть незавершённые» |
| Сброс таймера | Нет started команд | ❌ «Есть команды на дистанции» |
