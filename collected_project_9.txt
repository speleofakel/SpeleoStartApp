# ТЕХНИЧЕСКОЕ ЗАДАНИЕ v4.1
## Подземное ориентирование · СпелеоСтарт

---

## РАЗДЕЛ 1: ГЛОССАРИЙ (18 терминов)

### 1.1. Соревнование (Competition)
Отдельное мероприятие по спортивному ориентированию.

**Свойства:**
- `name` — полное название (для протоколов)
- `shortName` — краткое название (для статус-бара)
- `date`, `place`, `discipline`, `system`, `settingsJson`
- Жизненный цикл: СОЗДАНО → АКТИВНО → ЗАВЕРШЕНО → АРХИВИРОВАНО → УДАЛЕНО (и ОТМЕНЕНО)

### 1.2. Персона (Person)
Физическое лицо в общей базе. ФИО, дата рождения, пол, телефон, email, заметка, чёрная метка, никнейм, `canBeMentor`.

### 1.3. Участник (Participant)
Роль персоны в команде. Статусы: `active`, `replaced`, `free_agent`. Роли: `captain`, `member`.

### 1.4. Свободный агент (Free Agent)
Участник без активной команды. Может быть добавлен в другую.

### 1.5. Команда (Team)
Группа участников (от 2). Статусы: `registered`, `started`, `finished`, `disqualified`, `lost`.
Цветовые метки: `18+`, `14+`, `<14ок`, `<17 !!!`, `<14 !!!`, `СНЯТЫ`.

### 1.6. Класс сложности (Class)
2-й класс (КВ 90 мин), 3-й класс (КВ 60 мин). Разные наборы КП.

### 1.7. КП (Checkpoint)
Точка на дистанции. Номер, вес, тип (обычный/технический), норматив, бонус.

### 1.8. Эталонный маршрутный лист (Master Route Card)
Конфигурация всех КП из настроек.

### 1.9. Путевой лист (Team Route Card)
Фактические отметки команды по КП. Подтверждается секретарём и судьёй.

### 1.10. Старт (Start)
Момент начала движения. Интервал из настроек, чередование 2→3→2→3, первая команда 10 сек.

### 1.11. Финиш (Finish)
Время финиша = момент открытия диалога. Множественный финиш. Ручная корректировка.

### 1.12. Главный таймер (Main Timer)
`ММ:СС.сс`, от первого старта. Не останавливается. Восстанавливается после краша через `app_settings`.
### 1.13. Обратный отсчёт (Countdown Timer)
До старта текущей команды. Звук 800/1000 Гц.

### 1.14. КВ (Control Time)
90 мин (2-й класс), 60 мин (3-й класс). Просрочка → `lost`.

### 1.15. ПСР (Search & Rescue)
Статус `lost`, флаг `psrResolved`.

### 1.16. Откат старта (Rollback)
Возврат в `registered`. Если единственная на дистанции → сброс таймера.

### 1.17. Настройки соревнования (Competition Settings)
JSON с параметрами: интервал, КВ, КП, кастомные поля.

### 1.18. Ментор (Mentor)
Совершеннолетний (≥18), ответственный за участника <18. Может быть участником другой команды.

## РАЗДЕЛ 2: СУЩНОСТИ И СВЯЗИ

### Основные связи
| Связь | Тип | Внешний ключ |
|-------|-----|-------------|
| COMPETITION → TEAM | 1:N | `team.competitionId` |
| TEAM → PARTICIPANT | 1:N | `participant.teamId` |
| PERSON → PARTICIPANT | 1:N | `participant.personId` |
| PERSON → MENTOR | 1:1 | `mentor.personId` |
| MENTOR → PARTICIPANT | 1:N | `participant.mentorId` |
| COMPETITION → MASTER_ROUTE_CARD | 1:N | `master_route_card.competitionId` |
| MASTER_ROUTE_CARD → TEAM_ROUTE_CARD | 1:N | `team_route_card.checkpointId` |

### Правила целостности
- **R1:** Не стартовать без мин. участников
- **R2:** Одна персона не дважды в одной команде
- **R3:** Персона активна только в одной команде соревнования
- **R4:** Ментор ≥18 лет
- **R5:** Не редактировать подтверждённый ПЛ

## РАЗДЕЛ 6: КОМАНДЫ

### Статусы
`registered` → `started` → `finished`  
Любой → `disqualified`  
`started` → `lost` → (учёт ожидания) → `finished`

### Цветовые метки
- `18+` — все ≥18
- `14+` — есть 14-17, ментор подтверждён
- `<14ок` — есть <14, ментор + разрешение
- `<17 !!!` — есть 14-17, ментор НЕ подтверждён
- `<14 !!!` — есть <14, НЕТ разрешения
- `СНЯТЫ` — `disqualified`

### Нумерация
Сквозная в рамках класса и соревнования.

### Расформирование
- `registered` → без пароля, участники → `free_agent`
- `started` → с паролем + причина + возврат в `registered`
- `finished/lost/disqualified` → запрещено

---

## РАЗДЕЛ 7: СТАРТ

### Очередь
- Из `registered` команд
- Чередование 2→3→2→3
- Если класс пуст — подряд оставшиеся
- Пустая очередь → отсчёт остановлен, ждёт

### Таймеры
- Главный: `System.currentTimeMillis() - startTimestamp`
- Обратный: 10 сек (первая), интервал из настроек (остальные)
- Восстановление через `app_settings`

### Кнопки- НАЧАТЬ СОРЕВНОВАНИЯ → 10-сек отсчёт
- СНЯТЬ, ПРОПУСК, СТАРТ
- ПАУЗА/ПУСК

### Drag-and-drop очереди
Запрещён при ≤10 сек до старта.

## РАЗДЕЛ 9: КП И КАРТОЧКА

### Эталонный МЛ
- Настраивается в настройках соревнования
- Поля: `displayNumber`, `weight`, `type`, `sortOrder`, `normativeSeconds`, `class2/3`, `trackWait`, `trackExec`, `bonusPoints`

### Путевой лист
- Таблица: №, вес, тип, взят, ошибка, отсечка, штраф
- Подтверждение: секретарь + судья
- После двойного подтверждения → `read-only`

### Отсечка
- Время ожидания на тех. КП
- Вычитается из чистого времени
- `net_time = (finish - start) - Σ(offset_time)`

### Пересчёт `lost` → `finished`
Если чистое время ≤ КВ после учёта отсечек.

### CompetitionSettingsScreen

**Маршрут:** `competition_settings/{competitionId}`

**Назначение:** Просмотр и редактирование параметров соревнования, управление КП.

**Входные данные:** `competitionId: Long` (из аргументов навигации)

**ViewModel:** `CompetitionSettingsVM`

**Структура:**
1. TopAppBar — название (shortName → name fallback), «Назад», «Сохранить»
2. Блок «Контрольные пункты» — кластеры КП в рамках по классам, кнопка «Редактировать КП»
3. Блок «Параметры соревнования» — 4 числовых поля
4. Кнопка «СОХРАНИТЬ ВСЕ НАСТРОЙКИ»

**Особенности:**
- КП фильтруются по `forClass2` / `forClass3`
- Сортировка по `displayNumber`
- Клик по кластеру → диалог смены типа
- При пустом списке → подсказка + кнопка «Добавить КП»
- Автообновление КП при возврате с CheckpointListScreen (Flow)
- Сохранение в `settingsJson`: `{"start_interval":60,"control_time_2":90,"control_time_3":60,"min_team_size":2}`

**Навигация:**
- Вход: `CompetitionListScreen` (долгий тап или ⚙️)
- Выход 1: `CheckpointListScreen` (кнопка «Редактировать КП»)
- Выход 2: Назад → `CompetitionListScreen`

# 🏆 СпелеоСтарт: Инструкция судьи
*Коротко. По шагам. Без технических терминов.*

## 🔹 Быстрый старт (3 минуты)
1. Откройте приложение → «Соревнования» → «➕ Новое»
2. Введите название, дату, место → «СОЗДАТЬ»
3. Добавьте участников через «Персоны» → «➕ Новая персона»

### Экран «Настройки соревнования»

**Как открыть:** На экране списка соревнований нажмите и удерживайте карточку соревнования, или нажмите кнопку ⚙️.

**Блок «Контрольные пункты»:**
- КП отображаются круглыми кластерами: зелёные — обычные, оранжевые — технические
- Нажмите на кластер, чтобы изменить тип КП (обычный ↔ технический)
- Нажмите «Редактировать КП», чтобы перейти к полному редактированию списка

**Блок «Параметры соревнования»:**
- Интервал старта (секунды) — время между стартами команд
- Контрольное время 2-го класса (минуты)
- Контрольное время 3-го класса (минуты)
- Минимальный состав команды — минимальное число участников

**Сохранение:** Нажмите «💾 Сохранить» вверху или кнопку «СОХРАНИТЬ ВСЕ НАСТРОЙКИ» внизу. При ошибке валидации появится сообщение.


## 🔹 Зарегистрировать команду
1. Меню «Регистрация» → выберите класс
2. Введите фамилию:
    - ✅ Нашлось → выберите из списка
    - ❌ Нет → заполните прямо здесь:
      • Фамилия
      • Имя
      • Дата рождения (ДД.ММ.ГГГГ)
      > 💡 Поля «Отчество», «Телефон», «Позывной» скрыты. Нажмите «▸», чтобы заполнить сейчас, или сделайте это позже в полной карточке.
3. Если участнику <18 лет:
    - Приложение запросит ментора (≥18)
    - ✅ Нашлось → выберите
    - ❌ Нет → «➕ Быстрый ментор»: введите только Фамилию и Имя
   > 📌 Ментор привязывается к участнику. Если он уже в команде — просто выберите его.
4. Нажмите «СОХРАНИТЬ КОМАНДУ»

## 🔹 Начать старты
1. Меню «Старт» → «НАЧАТЬ СОРЕВНОВАНИЯ»
2. Очередь чередуется (2-й → 3-й класс) автоматически
3. Команда уходит по сигналу. Следующая появится сама

## 🔹 Управление таймером (НОВОЕ)
- **Остановка таймера:** Долгий тап по главному таймеру → введите пароль `1234` → таймер сбросится
- ⚠️ Используйте только в экстренных случаях!

## 🔹 Зафиксировать финиш
1. Нажмите 🏁 в нижнем углу
2. Отметьте галочками финишировавшие команды
3. Нажмите «ПОДТВЕРДИТЬ»
> ⏱️ Время фиксируется в момент открытия окна.

## 🔹 Скорректировать время финиша (НОВОЕ)
1. Откройте экран «Команды»
2. Найдите финишировавшую команду
3. **Долгий тап по карточке команды**
4. Введите правильное время в формате ЧЧ:ММ:СС
5. Нажмите «СОХРАНИТЬ»
> ⏱️ Время финиша обновится. Используется, если время было зафиксировано с ошибкой.


## 🔹 Заполнить путевой лист
1. Карточка команды → 📄 (или меню «Путевые»)
2. Для каждого КП: отметьте «Взят» / «Ошибка»
3. Для технических КП: укажите время ожидания
4. Введите пароль секретаря → «Карту заполнил»
5. Введите пароль судьи → «Карту проверил»
> 📌 После двойного подтверждения правки закрыты.
> ⚠️ Если в настройках отключено подтверждение: сохранение идёт сразу. Будьте внимательны.

## 🔹 Посмотреть итоги
1. Меню «Итоги» → выберите класс
2. Таблица: место, номер, баллы, время
3. Команды с пометкой `ОБГ` идут в конце списка. Их результаты считаются автоматически.

# 🛡️ VALIDATION_RULES.md · Правила валидации данных
**Дата:** 26.04.2026 | **Статус:** Рабочая | **Язык:** RU


## 1. ВВОД И ФОРМАТИРОВАНИЕ

| Поле | Правило | Где применяется |
|------|---------|-----------------|
| Фамилия, Имя, Отчество | 3 раздельных поля. **Title Case с явной русской локалью (`Locale("ru", "RU")`)**. Поддержка дефисных фамилий (Салтыков-Щедрин). Пробелы по краям обрезаются. | Регистрация персоны, инлайн-команда, быстрый ментор, редактирование |
| Дата рождения | Маска `ДД.ММ.ГГГГ`. Только цифры + авто-точки. Точная проверка возраста (с учётом дня/месяца). | Все формы ввода возраста |
| Телефон | Маска `+7 (XXX) XXX-XX-XX`. Валидация длины = 11 цифр. | Карточка персоны |
| Поиск персон | Мин. 3 символа фамилии ИЛИ 4 цифры телефона. Нормализация `Ё→Е`. | Автоподстановка в команде, менторе, замене |
| Поиск персон с созданием | Поле поиска + две кнопки: "+" для создания новой персоны, "✖" для очистки. Кнопка "+" активна только при непустом поле. Кнопка "✖" появляется только при наличии текста. | Экран регистрации команды |


**Технические детали Title Case:**
- Запрещено использовать `Locale.getDefault()` — на устройствах с английской локалью русские буквы обрабатываются некорректно
- Обязателен `Locale("ru", "RU")` для всех вызовов `uppercase()`/`lowercase()`
- Дефис (`-`) сбрасывает `capitalizeNext` для поддержки составных фамилий
- Пробел и точка также сбрасывают `capitalizeNext` (инициалы не ломаются)

| Возраст участника | Требование | Флаг в БД |
|-------------------|------------|-----------|
| ≥18 лет | Ментор не требуется | `mentorId = null` |
| 14–17 лет | Требуется ментор ≥18 + подтверждение | `mentorConfirmed = true` |
| <14 лет | Ментор ≥18 + подтверждение + разрешение судьи | `mentorConfirmed = true`, `judgeApproved = true` |

**Правила менторства:**
- 1 ментор → N подопечных в рамках одного соревнования.
- Ментор может входить в состав той же команды.
- Быстрая форма: только `Фамилия` + `Имя`. `canBeMentor=true` ставится автоматически. Дата рождения заполняется позже.
- В истории персоны: справка `[Соревнование] — ментор: [ФИО]`.

## 3. СОСТАВ КОМАНДЫ И ОБГ
- `min_team_size` читается из `competition.settings_json` (дефолт 2).
- Старт блокируется, если `участников < min_team_size`.
- Если `участников > min_team_size` → автоматическая метка `ОБГ`.
- ОБГ сортируется **после** стандартных зачётов в итогах. Ручная корректировка не требуется.

## 4. СТАТУСНЫЕ ПЕРЕХОДЫ (БЛОКИРОВКИ)
| Действие | Условие | Результат |
|----------|---------|-----------|
| Старт | `статус = registered` И `участников ≥ min` И `менторы подтверждены` | ✅ `started` |
| Финиш | `статус = started` | ✅ `finished` + фиксация времени |
| Расформирование | `registered` | ✅ без пароля → `free_agent` |
| Расформирование | `started` | ✅ пароль + причина → `registered` |
| Расформирование | `finished / lost / disqualified` | ❌ Запрещено |
| Смена класса | `started ≤5 мин` | ✅ пароль + штраф |
| Смена класса | `started >5 мин / finished` | ❌ Только откат/дисквалификация |

## 5. ПУТЕВОЙ ЛИСТ И ПОДТВЕРЖДЕНИЕ
- **Настройки соревнования:**
  - `Старший судья` (Да/Нет) → «Нет» только после пароля разработчика.
  - `Секретарь` (Да/Нет) → отключение только после пароля судьи.
- **Поведение:**
  - Если любой флаг `Нет` → показывать `⚠️ Сохраняется без подтверждений — будьте внимательны`.
  - Если оба `Да` → обязательный двойной ввод паролей. После → `checkpointsEntered = true`, режим `read-only`.
  - `Взят + Ошибка` → вес = 0, отсечка учитывается в чистом времени.
  - `Не взят` → вес = 0, отсечка не учитывается.
  - Автопересчёт: если `статус = lost` И `чистое_время ≤ КВ` → `статус = finished`.

## 6. ТЕХНИЧЕСКИЕ ОГРАНИЧЕНИЯ
| Область | Правило |
|---------|---------|
| БД | `version++` строго при изменении любого `@Entity`. Порядок очистки: дети → родители. |
| ID | Запрещён хардкод `1L`. Передача только через `SharedState` или `navArgument`. |
| Insert | Всегда проверять `insert() != -1L`. При `-1L` → лог + возврат, без продолжения транзакции. |
| UI | Мин. тап 48dp. Цвета только через `MaterialTheme.colorScheme` или явную проверку светлой/тёмной темы. |
| Таймер | Фиксация финиша при открытии диалога. Восстановление: `System.currentTimeMillis() - startTimestamp` из `app_settings`. |
